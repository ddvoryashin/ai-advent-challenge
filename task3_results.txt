Write an algorithm to determine whether a string is a palindrome, ignoring spaces, punctuation and case. Examples: "A man, a plan, a canal: Panama" -> true, "race a car" -> false.

Here's an algorithm to determine if a string is a palindrome while ignoring spaces, punctuation, and case:

## Algorithm

**Input**: A string `s`
**Output**: Boolean (true if palindrome, false otherwise)

### Steps:

1. **Initialize two pointers**:
   - `left` = 0 (start of string)
   - `right` = length of `s` - 1 (end of string)

2. **While `left < right`**:
   - **Skip non-alphanumeric characters from left**: 
     - While `left < right` AND `s[left]` is NOT alphanumeric, increment `left`
   - **Skip non-alphanumeric characters from right**:
     - While `left < right` AND `s[right]` is NOT alphanumeric, decrement `right`
   
   - **Compare characters**:
     - Convert both `s[left]` and `s[right]` to lowercase
     - If they are NOT equal, return `false`
   
   - **Move pointers inward**:
     - Increment `left`
     - Decrement `right`

3. **Return `true`** (all characters matched)

### Pseudocode:

```
function isPalindrome(s):
    left = 0
    right = s.length - 1
    
    while left < right:
        while left < right AND not isAlphanumeric(s[left]):
            left++
        
        while left < right AND not isAlphanumeric(s[right]):
            right--
        
        if toLowerCase(s[left]) != toLowerCase(s[right]):
            return false
        
        left++
        right--
    
    return true

function isAlphanumeric(c):
    return (c is letter 'a'-'z' or 'A'-'Z') OR (c is digit '0'-'9')
```

3. META-PROMPT -> SOLUTION
============================================================
Here's a solution in Python, which is clear and efficient:

```python
def is_palindrome(s: str) -> bool:
    """
    Determines whether a given string is a palindrome,
    ignoring spaces, punctuation, and case.
    """
    # Step 1: Normalize the string
    # - Convert to lowercase
    # - Keep only alphanumeric characters (letters and digits)
    normalized = ''.join(ch.lower() for ch in s if ch.isalnum())

    # Step 2: Check if the normalized string equals its reverse
    # Using two-pointer approach for O(n) time and O(1) extra space
    left = 0
    right = len(normalized) - 1

    while left < right:
        if normalized[left] != normalized[right]:
            return False
        left += 1
        right -= 1

    return True
```

**Explanation of the logic:**

1. **Normalize the input:**  
   - Convert all characters to lowercase so we ignore case.  
   - Filter out anything that isn't a letter or digit (e.g., spaces, commas, colons). This leaves only the core characters to compare.

2. **Two-pointer comparison:**  
   - Use two pointers: `left` starts at the beginning, `right` at the end.  
   - Compare characters at each pointer position.  
   - If they ever differ, it's not a palindrome → return `False`.  
   - Move the pointers inward until they meet (the middle of the string).  
   - If all comparisons match, return `True`.

**Example runs:**
- `is_palindrome("A man, a plan, a canal: Panama")` → `True`  
- `is_palindrome("race a car")` → `False`

**Time complexity:** O(n) — single pass over the string to normalize and another pass for comparison.  
**Space complexity:** O(n) due to the normalized copy. The two-pointer part uses O(1) extra space.

ANALYST:
A robust algorithm:
1. Initialize two pointers, left = 0, right = string length - 1.
2. While left < right:
   - While left < right and character at left is not alphanumeric, increment left.
   - While left < right and character at right is not alphanumeric, decrement right.
   - Compare lowercased characters at left and right:
     - If not equal, return false.
   - Increment left, decrement right.
3. Return true.

Edge cases: empty string → true; single character → true; null input → handle as needed. Time O(n), space O(1).

ENGINEER:
Clean, idiomatic Python:

```python
import re

def is_palindrome(s: str) -> bool:
    # Filter and normalize in one step
    cleaned = re.sub(r'[^a-zA-Z0-9]', '', s).lower()
    return cleaned == cleaned[::-1]
```

This is concise, readable, and leverages Python's string slicing. For very large strings, the two-pointer approach (ANALYST) is more memory efficient, but this is the most Pythonic.

CRITIC:
Flaws in both:
- ANALYST's two-pointer is correct but verbose and harder to read. It also assumes ASCII; non-ASCII characters (e.g., "café") may not be handled correctly without Unicode support.
- ENGINEER's solution creates a new string, which is O(n) memory. This is inefficient for large inputs. Also, `re.sub` adds overhead; a list comprehension with `str.isalnum()` is faster.

Better compromise:

```python
def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        if not s[left].isalnum():
            left += 1
        elif not s[right].isalnum():
            right -= 1
        elif s[left].lower() != s[right].lower():
            return False
        else:
            left += 1
            right -= 1
    return True
```

This is O(n) time and O(1) space, handles Unicode alphanumeric characters properly, and is clean enough to read. The ENGINEER's approach is acceptable only for small strings or rapid prototyping.
